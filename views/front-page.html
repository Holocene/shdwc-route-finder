Summary

This is a tool for keeping track of wormholes, and also route finding through them. It is still in development, and is only tested in Firefox and Chrome.

Check out the 'route finder' tab, which is the most interesting thing in the system right now.

There are lots of features left to add. Two that I can think of are the Distance Display and Automatic deletion of expired wormholes, but I'm sure there is a bunch more that could be done. If you have any suggestions, post them to the forums, add an issue in the issue tracker or contact me directly: Olixia Castitatis. All suggestions are appreciated, but if you have an idea for an extensive new feature, it's best if you try to set out exactly how you imagine it working, as I tried to do in my initial post on the forums here .

The site is developed using javascript, html and css. Angular JS provides the framework in which the site is built. The site is hosted by Github, and the database is provided by Firebase. Some backend services are provided by Heroku.

How do I use this tool?

Scanning

  1. Go to the systems page
  2. Find the system you are in.
  3. In EVE, scan some signatures with your probe scanner. Select them all, then copy them to clipboard (hint: ctrl-A, ctrl-C).
  4. Click 'paste scan results' to copy your results into the tool
  5. Add any wormholes you find.

If you add the signatures you find to the tool, it's easier for the next prober to work out what sigs are new and what have stayed the same.

Route finding

Go to the route finding page. Enter your start and destination system and you'll see an array of routes. Use the filters on the right to set system restrictions (no highsec) or transport method restrictions (no titans, or no wormholes).
